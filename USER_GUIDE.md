## 1. Назначение приложения

Приложение помогает менеджерам проектов:

- планировать сроки задач и вех;
- отслеживать статус, стоимость и трудозатраты;
- визуализировать дорожную карту в нескольких форматах (Gantt, таймлайн, Kanban, таблица, бэклог, релизы);
- вести историю изменений с указанием автора;
- группировать задачи по категориям (столбец **Область** в таблице), релизам и целям;
- связывать несколько «использований» одного источника данных с общими сроками и этапами;
- **импортировать проекты** из Excel или JSON прямо со стартовой страницы;
- **задавать вопросы об открытом проекте** через ИИ-ассистент на GigaChat (сроки, статусы, стоимость, сдвиги этапов).

Демо-проект **«Витрины данных»** создаётся автоматически из файла `data/DataMarts.xlsx` при первом запуске.

## 2. Первый запуск

1. Запустите приложение (см. [README.md](README.md#быстрый-старт)).
2. Откройте в браузере **http://localhost:8080** (именно порт **8080**).
3. Введите **отображаемое имя** — оно будет записываться в журнал аудита и комментарии.
4. На стартовой странице выберите проект, **импортируйте файл** или создайте новый, затем нажмите **Открыть проект**.

Имя можно изменить только очистив локальные данные браузера (оно хранится локально).

## 3. Обзор интерфейса

После открытия проекта экран делится на три зоны.

### Верхняя панель (toolbar)

| Элемент | Назначение |
|---------|------------|
| **← Проекты** | Возврат на стартовую страницу |
| Название проекта | Текущий открытый проект |
| Бейдж с именем | Ваше имя для аудита |
| Вкладки представлений | Гант · Таймлайн · Канбан · Таблица · Бэклог · Релизы |
| **Индикативные сроки** | Показать/скрыть пунктирные индикативные диапазоны |
| **Комментарии к сдвигам** | Показать/скрыть текст комментариев к переносам дат |
| Группировка | **Цветовая кодировка** или **Группы по категориям** (swimlane) |
| **Категории** | Управление категориями (отображаются в таблице как **Область**) |
| **Источники** | Список общих источников данных |
| **Релизы** | Управление релизами |
| **Цели** | Управление целями проекта |
| **Журнал аудита** | История изменений по всему проекту |

### Основная область

Отображает выбранное представление (Gantt, таблица и т.д.).

### Боковая панель задачи (drawer)

Открывается при клике на задачу в любом представлении. Содержит все свойства задачи, этапы, комментарии и историю.

### ИИ-ассистент (чат)

В **правом нижнем углу** — круглая кнопка **💬**. Открывает панель чата с ассистентом по **текущему проекту**. Подробнее — в [разделе 17](#17-ии-ассистент).

### Нижняя панель сохранения

Появляется автоматически, когда есть несохранённые локальные изменения.

## 4. Проекты

### Стартовая страница

- Список всех проектов с краткой статистикой: число задач, категорий и задач «В работе».
- **+ Новый проект** — форма с названием и необязательным описанием.
- **Импорт из файла** — зона перетаскивания (drag-and-drop) или выбор файла; поддерживаются **.xlsx**, **.xls**, **.json**.
- Выберите проект в списке и нажмите **Открыть проект**.

### Импорт проекта из файла

На стартовой странице перетащите файл в зону импорта или нажмите для выбора файла. Укажите название проекта (по умолчанию — имя файла) и при необходимости описание, затем нажмите **Создать проект из файла**.

| Формат | Поведение |
|--------|-----------|
| **DataMarts** (столбцы БВ, Источник, Субпродукт и др.) | Полный импорт как у демо-проекта: категории, общие источники, этапы |
| **Произвольная таблица** | Заголовки сопоставляются с известными полями (название, статус, даты, категория…); остальные сохраняются как **дополнительные столбцы** |
| **JSON** | Структурированный экспорт/импорт проекта |

После импорта откройте проект — в представлении **Таблица** отобразятся столбцы, соответствующие загруженным данным.

> **Совет:** если в файле есть столбцы **БВ**, **Источник** и **Субпродукт**, приложение автоматически применит логику импорта витрин данных (включая объединение по источнику).

### Демо-проект «Витрины данных»

Создаётся при первом запуске из Excel-файла. Содержит:

- категории БВ (бизнес-витрины);
- строки «использований» витрин;
- общие компоненты по полю **Источник** (один источник — несколько использований с синхронизированными сроками).

Чтобы обновить данные после правки `data/DataMarts.xlsx`, выполните команду повторного импорта из README.

## 5. Представления

Все представления работают с **одними и теми же данными**. Переключение вкладок не теряет несохранённые изменения — они остаются в локальном буфере до нажатия **Сохранить**.

### Гант

**Для чего:** наглядная временная шкала с полосами задач, зависимостями и вехами.

**Основные действия:**

| Действие | Как выполнить |
|----------|---------------|
| Переместить задачу | Перетащите полосу задачи по горизонтали |
| Изменить длительность | Потяните за правый край полосы |
| Открыть свойства | Кликните по названию задачи слева или по полосе |
| Переместить веху | Перетащите ромб вехи на шкале |
| Свернуть группу | В режиме swimlane кликните заголовок группы (▸/▾) |

**Дополнительно:**

- **Показывать приоритет** — бейджи P1, P2… и цветовая полоска на полосе.
- **Фильтр по приоритету** — скрыть задачи с нежелательными приоритетами.
- Сплошная полоса — **фактические (обязательные) сроки**; пунктирная обводка — **индикативный** диапазон (если включён переключатель).
- Под основной полосой задачи — **зелёная полоса выполненного этапа** (фактические даты завершённых этапов) — видна при **развёрнутой** шкале этапов (см. ниже).
- **▸ / ▾ у задачи с этапами** — развернуть или свернуть подстроки этапов. По умолчанию этапы **свёрнуты**; при разворачивании под задачей появляются строки этапов с полосами дат и **фиолетовыми стрелками зависимостей** между ними.
- **Фиолетовые пунктирные стрелки** между подстроками этапов — зависимости (от окончания предшественника к началу последователя). Настраиваются в drawer → **Этапы** → блок **Связи с этапами** у каждого этапа. Видны **только в развёрнутом** виде.
- Оранжевая обводка и звёздочка `*` — несохранённые изменения.
- Серые стрелки между строками — зависимости **между задачами** (finish-to-start и другие типы).
- **Дугообразные стрелки сдвига** (красные — позже, зелёные — раньше) над или под полосой: сдвиг задачи, индикатива или этапа; при нескольких сдвигах дуги разводятся по высоте, чтобы не накладываться.
- Зелёная галочка и зачёркнутый текст — задача выполнена на 100 %.

### Таймлайн

**Для чего:** хронологический список задач по категориям с редактируемыми датами.

**Основные действия:**

- Вверху — полоска **вех** с датами.
- Каждая задача — карточка с полями **начала** и **окончания** (можно менять прямо в списке).
- Клик по карточке открывает боковую панель.
- В режиме swimlane заголовки секций сворачиваются; в свёрнутом виде показываются агрегированные диапазоны дат группы.

### Канбан

**Для чего:** управление статусом задач.

**Колонки:** Не начато · В работе · На паузе · Готово.

**Основные действия:**

- Перетащите карточку в другую колонку — статус изменится локально (нужно сохранить).
- На карточке: исполнитель, процент выполнения, сроки, индикаторы сдвига дат.
- Вверху — ближайшие вехи проекта.

### Таблица

**Для чего:** массовое редактирование в формате электронной таблицы.

**Основные действия:**

| Действие | Как выполнить |
|----------|---------------|
| Редактировать ячейку | Кликните, измените значение, уберите фокус (Tab или клик вне поля) |
| Выбрать строку | Кликните по строке (не по полю ввода) — строка подсветится синим |
| Открыть drawer | Клик по строке (кроме input/select/textarea/button) |
| Добавить задачу | **+ Добавить задачу** |
| Удалить задачу | Выберите строку → **Удалить задачу** |
| Свернуть группу | В режиме swimlane кликните строку-заголовок группы |

**Основные столбцы:** приоритет, статус, **Область** (категория), использование (название), источник, **Витрина**, формы, заказчик, площадка, даты (факт и индикатив), длительность, % выполнения, риски, комментарий, стоимость, трудозатраты, предшественники.

**Адаптивные столбцы:** набор колонок зависит от проекта. Для импортированных таблиц с нестандартными заголовками появляются дополнительные столбцы; пустые колонки скрываются автоматически. Стандартные поля (даты, статус, категория и т.д.) отображаются, если в данных есть хотя бы одно непустое значение.

Бейдж **Общий** у источника означает, что задача связана с общим компонентом и используется в нескольких строках.

### Бэклог

**Для чего:** приоритизация и сортировка задач.

**Методы приоритизации** (переключатель вверху):

| Метод | Поля | Формула / логика |
|-------|------|------------------|
| **RICE** | Охват, Влияние, Уверенность %, Усилие | (Охват × Влияние × Уверенность/100) / Усилие |
| **Ценность / Усилие** | Ценность, Усилие | Ценность / Усилие |
| **MoSCoW** | Must / Should / Could / Won't | Весовая шкала (Must выше всех) |

Список автоматически сортируется по убыванию оценки. Поля можно редактировать прямо в таблице. Клик по строке открывает drawer.

### Релизы (release board)

**Для чего:** распределение задач по релизам.

- Колонка на каждый релиз + колонка **Без релиза**.
- Перетащите карточку задачи между колонками — назначение релиза изменится локально.
- В заголовке колонки: статус релиза (Запланирован / В работе / Выпущен) и целевая дата.

## 6. Редактирование задач

Задачу можно менять тремя способами — изменения объединяются в один локальный буфер.

### В таблице

Быстрое массовое редактирование: статус, категория, даты, стоимость, предшественники и др.

### На диаграмме Ганта

Перетаскивание полос и изменение длительности правым краем. Подходит для визуального планирования сроков.

### В боковой панели (drawer)

Полный набор полей, разбитый на **вкладки**:

**Общая**

- Описание задачи, приоритет
- **Витрина**, **Источник**, **Формы**, **Заказчик**, **Площадка**, **Область** — выпадающие списки с возможностью ввода: можно выбрать ранее использованное в проекте значение или ввести новое (после сохранения оно появится в подсказках у других задач). У задачи с привязанным общим источником поле **Источник** только для чтения
- **Желаемый срок** — календарь (дата)

**Остальное**

- Планирование: релиз, цель, MoSCoW, RICE, Ценность/Усилие
- Фактические и индикативные сроки, риски, комментарий к задаче
- Полоса прогресса выполнения

**Подрядчик**

- Подрядчик, плановая и фактическая стоимость

**Трудозатраты**

- Количество атрибутов, калькулятор плановых трудозатрат, плановые и фактические трудозатраты

**Этапы**

- Список подэтапов с **нумерацией** (1., 2., 3. …) по порядку в списке
- Даты **Начало** и **Окончание**; у каждого этапа — блок **Связи с этапами** (добавление связи, тип **После** / **До** для каждой пары, удаление)
- Блок **Зависимости от других задач** — связи с задачами проекта (см. [раздел 9](#9-зависимости-между-задачами))
- Для каждого **незавершённого** этапа — кнопки **Выполнено**, **Сдвиг** и **Удалить**, даты и **Связи с этапами** (подробнее в [разделе 12](#12-этапы-и-прогресс-выполнения))
- **Отметить все выполненными**
- **+ Добавить этап** — выбор из **шаблона** (стандартные этапы витрин, ранее сохранённые в проекте), **набора этапов** (массовое добавление) или **свой этап** с произвольным названием; при добавлении одного этапа можно указать **связи с существующими этапами** и для каждой выбрать тип **После** или **До**; даты **Начало** / **Окончание** — при заданном начале окончание не раньше следующего дня
- **Сохранить в шаблоны проекта для повторного использования** — чтобы использовать тот же этап в других задачах
- **Индикативные сроки** задачи при наличии этапов считаются автоматически (min начало, max окончание по этапам)

**Комментарии и история**

- Лента комментариев с автором и временем
- Фильтруемая история изменений по типу события

**Удаление**

- Кнопка **Удалить задачу** внизу drawer (с подтверждением)

> **Примечание:** изменения этапов (выполнение, сдвиг, добавление, удаление) сохраняются на сервер **сразу**, без панели «Сохранить». Остальные поля задачи — через локальный буфер.

## 7. Локальные изменения и сохранение

Приложение использует **отложенное сохранение**: правки накапливаются локально, пока вы не нажмёте **Сохранить изменения**.

### Как понять, что есть несохранённые изменения

- Оранжевая подсветка строк/карточек/полос
- Звёздочка `*` рядом с названием задачи
- Бейдж **не сохранено** в drawer
- Панель внизу экрана с перечнем изменённых задач

### Панель сохранения

Появляется внизу, когда есть хотя бы одно локальное изменение.

| Кнопка | Действие |
|--------|----------|
| **Отменить** | Сбросить все локальные изменения |
| **Сохранить изменения без сдвига** | Записать в БД, но не фиксировать сдвиги дат в визуальных индикаторах |
| **Сохранить изменения** | Записать в БД и зафиксировать сдвиги; комментарии к сдвигам и к смене статуса (из подсказки по активному этапу) сохраняются как комментарии к задаче |

### Конфликт версий

Если другой пользователь изменил ту же задачу, при сохранении может появиться ошибка **Конфликт версий**. Обновите страницу, чтобы получить актуальные данные, и повторите правку.

### Оптимистическая блокировка

Каждая задача имеет поле `version`. При сохранении сервер проверяет, что версия не изменилась с момента загрузки.

## 8. Сдвиги дат и комментарии

Когда вы меняете даты задачи, вехи или **этапа**, приложение показывает **индикатор сдвига** на диаграмме Ганта:

- **Красная дугообразная стрелка** — срок сдвинулся позже (задержка)
- **Зелёная дугообразная стрелка** — срок сдвинулся раньше (ускорение)
- Пунктирная обводка — исходный диапазон до сдвига
- Подпись вида «+3d» / «−2d» на дуге; при нескольких сдвигах дуги размещаются **над или под** полосой на разных «слоях», чтобы не перекрываться

Сдвиги отображаются для:

| Объект | Когда фиксируется |
|--------|-------------------|
| **Задача** | При сохранении изменённых фактических дат через панель «Сохранить» |
| **Индикатив** | Когда пересчитанный индикативный диапазон изменился из‑за сдвига этапа или задачи |
| **Этап** | При сохранении **Сдвига** этапа или при завершении этапа с изменёнными датами |
| **Веха** | При сохранении переноса вехи на Gantt |

### Комментарий к сдвигу

**Задачи и вехи** (локальный буфер):

1. Кликните по индикатору сдвига (на Gantt, в таблице, drawer или Kanban).
2. Введите причину переноса: «Почему перенесли срок?»
3. При сохранении с основной кнопкой **Сохранить изменения** текст попадёт в комментарии задачи.

**Этапы** — комментарий **обязателен** при сдвиге (кнопка **Сдвиг** в drawer или изменение дат этапа в списке). Текст сохраняется в комментариях задачи и отображается на Gantt при включённом переключателе **Комментарии к сдвигам**.

В панели сохранения для задач со сдвигом дат появляются раскрывающиеся поля комментариев.

Переключатель **Комментарии к сдвигам** в toolbar показывает или скрывает текст комментариев прямо на диаграмме.

## 9. Зависимости между задачами

Зависимости определяют порядок **задач** в проекте. На Gantt они отображаются **серыми стрелками** между строками — от окончания предшественника к началу последователя.

> **Зависимости между этапами** одной задачи настраиваются на вкладке **Этапы** (блок **Связи с этапами** у каждого этапа и при добавлении нового) и показываются **фиолетовыми стрелками** между подстроками этапов на Gantt. См. [раздел 12](#12-этапы-и-прогресс-выполнения).

### Задание предшественников

**В drawer** (вкладка **Этапы** → блок **Зависимости от задач**): выберите задачу-предшественника, при необходимости укажите **этап предшественника** и **этап этой задачи**, тип связи (FS / SS / FF / SF).

**В таблице** — столбец **Предшественники**. Формат текста:

| Пример | Смысл |
|--------|--------|
| `Задача А` | Вся задача А → вся текущая задача (FS) |
| `Задача А:2` | Этап 2 задачи А |
| `Задача А>3` | → этап 3 текущей задачи |
| `Задача А:2>3` | Этап 2 задачи А → этап 3 текущей |
| `Задача А:2>3@SS` | То же, тип SS |

На **Gantt** зависимости с этапами отображаются **фиолетовой пунктирной стрелкой** (при развёрнутых этапах — между подстроками этапов); без этапов — серая стрелка между строками задач.

Если у зависимого **этапа** не задано начало, drawer предлагает автозаполнение: для типа **FS** — **день после окончания** указанного этапа предшественника (или задачи, если этап не выбран).

### Каскадный сдвиг

При изменении дат предшественника сервер **автоматически сдвигает последователей**, чтобы соблюсти тип зависимости:

| Тип | Описание |
|-----|----------|
| **FS** (finish-to-start) | Последователь начинается не раньше дня после окончания предшественника |
| **SS** (start-to-start) | Начала синхронизированы |
| **FF** (finish-to-finish) | Окончания синхронизированы |
| **SF** (start-to-finish) | Редкий тип; учитывается длительность предшественника |

Каскад выполняется на сервере в момент сохранения. Затронутые задачи обновятся после перезагрузки данных.

## 11. Общие источники данных

Один **источник** (например, одна система-источник данных) может использоваться в нескольких строках таблицы («использованиях»). Сроки, этапы и статус хранятся **один раз** и синхронизируются между всеми связанными задачами.

### Как это выглядит

- В таблице у источника — бейдж **Общий** и счётчик использований.
- В drawer — баннер **Общий источник** со списком других использований (ссылки для перехода).

### Окно «Источники»

Toolbar → **Источники**:

- Список всех общих компонентов проекта.
- **Создать источник** — укажите название системы-источника и нажмите кнопку (источник появится в списке; витрины можно привязать позже).
- Для каждого — перечень использований (клик открывает задачу).

### Создание и привязка из карточки витрины

На вкладке **Общая**, если витрина **ещё не привязана** к общему источнику:

1. Заполните поле **Источник** (название системы-источника данных).
2. **Сделать общим источником** — сроки, этапы и статус этой витрины переносятся в общий компонент; витрина становится первым использованием.
3. **Привязать к общему источнику** — если источник с таким именем уже есть в проекте.
4. Или выберите источник в списке и нажмите **Привязать**.

После привязки поле **Источник** только для чтения; сроки и этапы редактируются один раз и видны во всех связанных витринах.

### Что синхронизируется

Между всеми витринами, привязанными к одному источнику, общие:

| Данные | Где редактировать |
|--------|-------------------|
| Фактические и индикативные сроки | Любая привязанная витрина (вкладки **Общая**, Gantt, таблица) |
| Статус и % выполнения | Любая привязанная витрина |
| Этапы (добавление, сдвиг, выполнение, удаление) | Вкладка **Этапы** любой привязанной витрины |
| Исполнитель, подрядчик, площадка | Вкладка **Общая** / **Подрядчик** |

У каждой витрины остаются **свои**: название, витрина (субпродукт), категория, релиз, цель, стоимость и трудозатраты по задаче.

Изменения этапов и общих полей сохраняются на сервер **сразу** и отображаются у всех использований без отдельного сохранения.

### Отвязка

В drawer нажмите **Отвязать (сохранить копию)**:

- Задача получит локальную копию данных источника.
- Изменения больше не синхронизируются с другими использованиями.

### Импорт из Excel

Импорт возможен двумя способами:

1. **Стартовая страница** — перетащите `.xlsx`, `.xls` или `.json` в зону импорта (см. [Импорт проекта из файла](#импорт-проекта-из-файла)).
2. **Seed при запуске** — демо-проект «Витрины данных» из `data/DataMarts.xlsx`.

При загрузке DataMarts строки с одинаковым **Источником** автоматически объединяются в один общий компонент.

## 12. Этапы и прогресс выполнения

Каждая задача может содержать список **этапов** (подэтапов). У каждого этапа есть даты **Начало** и **Окончание**. Стандартные названия этапов взяты из шаблона витрин данных (`Template_substages`); в проекте можно накапливать **свои шаблоны** для повторного использования.

### Действия с этапом

У **незавершённого** этапа в drawer три кнопки:

| Кнопка | Назначение |
|--------|------------|
| **Выполнено** | Отметить этап завершённым. Откроется окно подтверждения с фактическими датами начала и окончания. Если даты скорректированы — потребуется комментарий; сдвиг отобразится на Gantt, индикативные сроки пересчитаются при необходимости. |
| **Сдвиг** | Перенести сроки этапа без завершения. Укажите новые даты и **обязательный** комментарий. Сдвиг фиксируется на Gantt дугообразной стрелкой под полосой этапа; если изменение влияет на общий диапазон этапов — обновятся **индикативные сроки** задачи. |
| **Удалить** | Безвозвратно удалить этап. Перед удалением показываются предупреждения о связанных зависимостях (внутри задачи и между задачами). |

Для **выполненного** этапа доступна только ссылка **Снять отметку о выполнении** — редактирование дат, связей и удаление скрыты до снятия отметки.

Даты незавершённого этапа можно также изменить в полях **Начало** / **Окончание** в списке — при уходе с поля откроется то же окно **Сдвига** с уже введённым значением.

**Отметить все выполненными** — завершить все этапы разом.

**+ Добавить этап** — форма с выпадающим списком шаблонов.

### Даты этапов и индикатив

- При указании **начала** у 2-го и следующих этапов приложение может предложить автоматически проставить **окончание предыдущего** этапа на день раньше.
- При добавлении нового этапа без начала — предложение взять **начало = окончание предыдущего + 1 день**.
- Если у этапа задано **начало**, при выборе **окончания** (в списке этапов, в форме **+ Добавить этап**, в окнах **Сдвиг** и **Выполнено**) календарь открывается с **дня после начала**; более ранние даты недоступны. Если окончание ещё пустое, подставляется этот день как стартовая точка выбора.
- **Индикативные сроки** задачи (поля в drawer) при наличии этапов **пересчитываются автоматически**: минимальное начало и максимальное окончание по всем этапам (с учётом `due_date`, если дата окончания не задана). Ручная правка индикатива возможна, пока нет этапов или до следующего изменения этапов.

На диаграмме **Гант** после завершения этапа под полосой задачи появляется **зелёная полоса** с фактическими датами этого этапа (в свёрнутом виде — полоски запланированных этапов; зелёный цвет у завершённых).

**Фактические сроки задачи** (полоса задачи, таблица, Timeline, Kanban) и **процент выполнения** считаются по **оставшимся выполненным** этапам. **Индикативные сроки** — по **запланированным** (`is_indicative`) этапам.

### Подсказка статуса при активном этапе

Если **запланированный** этап (уже на диаграмме Ганта / в индикативе) **не завершён** и его срок **включает сегодняшний день**, а статус задачи **не «В работе»**, приложение предлагает обновить статус:

| Когда появляется | Что происходит |
|------------------|----------------|
| При **входе в проект** | Окно показывается по очереди для всех подходящих задач |
| После **планирования этапа** или **изменения его дат** в drawer | Если этап стал активным сегодня |

В окне **Обновить статус задачи**:

- по умолчанию выбран статус **В работе**;
- **Применить** — статус попадает в локальные изменения (нужно **Сохранить изменения** внизу экрана);
- если выбран **любой другой** статус — **комментарий обязателен**; он сохранится в комментариях задачи при сохранении;
- **Позже** — подсказка для этой задачи не показывается до следующего дня.

> Подсказка не появляется, если задача уже в статусе «В работе» или этап выполнен.

### Удаление этапа

1. В drawer → **Этапы** нажмите **Удалить** у нужного **незавершённого** этапа (для выполненного сначала снимите отметку).
2. Прочитайте предупреждения: будут удалены или обновлены зависимости, ссылающиеся на этот этап.
3. Подтвердите удаление.

После удаления:

- этап исчезает из списка, с Gantt и из всех представлений;
- **фактические даты** задачи пересчитываются по оставшимся выполненным этапам (если их нет — сбрасываются);
- **индикативные сроки** пересчитываются по оставшимся запланированным этапам;
- **процент выполнения** обновляется;
- индикаторы сдвига дат для удалённого этапа снимаются.

> У общего источника удаление этапа в одной задаче удаляет его у **всех** связанных использований.

### Нумерация и зависимости между этапами

Этапы в drawer нумеруются **по порядку в списке** (1, 2, 3…). Номер не меняется при сортировке по датам — он соответствует позиции этапа в списке на вкладке **Этапы**.

Связи между этапами **одной задачи** задаются в блоке **Связи с этапами** у каждого этапа или при **добавлении нового этапа**:

| Тип связи | Смысл |
|-----------|--------|
| **После** | Текущий этап начинается после окончания выбранного этапа |
| **До** | Текущий этап должен завершиться до начала выбранного этапа |

Для каждой связи можно отдельно выбрать тип **После** или **До**. У существующего этапа: выберите другой этап, укажите тип и нажмите **+ Добавить связь**; тип можно изменить в списке или удалить связь. При добавлении нового этапа выберите один или несколько этапов в списке и задайте тип для каждого.

Изменения сохраняются на сервер **сразу**. Нельзя связать этап сам с собой или добавить дублирующую связь.

При связи типа **После**, если у зависимого этапа не задано **начало**, приложение может предложить дату **день после окончания** этапа-предшественника (как при межзадачных зависимостях).

На диаграмме **Гант** зависимости отображаются **фиолетовыми пунктирными стрелками** между подстроками этапов: стрелка идёт от **окончания** предшественника к **началу** зависимого этапа. Чтобы увидеть этапы и стрелки, нажмите **▸** слева от названия задачи (развернуть шкалу этапов). По умолчанию этапы свёрнуты. Для отображения у обоих этапов должны быть заданы даты. Наведите курсор на стрелку — подсказка покажет номера и названия этапов.

> Зависимости между этапами **не сдвигают даты автоматически** — они служат для визуализации порядка работ. Каскадный сдвиг по датам работает только для [зависимостей между задачами](#9-зависимости-между-задачами).

### Шаблоны этапов

В выпадающем списке **Этап из шаблона** четыре раздела:

| Раздел | Содержимое |
|--------|------------|
| **Наборы этапов** | Массовое добавление готовой цепочки этапов (см. ниже) |
| **Стандартные (Template_substages)** | Корпоративные этапы витрин (группы вроде «Детальный слой / Аналитика», «Витрина данных / …») |
| **Шаблоны проекта** | Этапы, сохранённые вами ранее в этом проекте |
| **Использованные ранее** | Этапы, уже добавленные в другие задачи этого проекта |
| **Свой этап…** | Произвольное название без шаблона |

Этапы, уже присутствующие на задаче, помечены в списке и недоступны для повторного добавления.

### Наборы этапов (массовое добавление)

В разделе **Наборы этапов** доступны два пресета:

| Набор | Что добавляется |
|-------|-----------------|
| **Загрузка детального слоя** | Все стандартные этапы группы «Детальный слой / …» (аналитика и разработка детального слоя) |
| **Разработка Витрины** | Все стандартные этапы группы «Витрина данных / …» (аналитика, разработка и публикация витрины) |

Как использовать:

1. Drawer → **Этапы** → **Этап из шаблона** → выберите набор (в скобках — число этапов, которые будут добавлены).
2. Нажмите **+ Добавить N этапов**.
3. На задачу попадут только этапы, которых ещё нет; уже существующие пропускаются.
4. Новые этапы автоматически связываются цепочкой **После** (каждый следующий — после предыдущего; первый новый — после последнего уже существующего этапа, если он есть).

Сроки при массовом добавлении не задаются — их можно указать позже через **Сдвиг**, **Запланировать** или поля дат в списке.

При добавлении **одного** этапа можно указать срок и отметить **Индикативный** для предварительных этапов.

**Сохранить в шаблоны проекта для повторного использования** — после ввода названия этапа сохраните его в библиотеку проекта; он появится в разделе «Шаблоны проекта» при добавлении этапов к другим задачам.

Индикативные этапы визуально отличаются в списке.

### Прогресс

- Полоса прогресса в drawer и процент на карточках Kanban.
- При 100 % — галочка и зачёркнутое название на Gantt и в Kanban.

> Этапы общего источника синхронизируются между всеми связанными задачами.

## 14. Приоритизация бэклога

Представление **Бэклог** сортирует задачи по выбранному методу.

### RICE

Заполните в drawer или в таблице бэклога:

- **Охват** — сколько людей/единиц затронет
- **Влияние** — сила эффекта (обычно 1–3)
- **Уверенность %** — насколько вы уверены в оценках
- **Усилие** — трудозатраты в условных единицах

Чем выше итоговый балл, тем выше задача в списке.

### Ценность / Усилие

Простая формула: отношение ценности к усилиям. Подходит для быстрой приоритизации без детальной RICE-оценки.

### MoSCoW

Классификация Must / Should / Could / Won't. Задачи с Must оказываются вверху списка.

## 15. Вехи

**Веха** — значимая точка на временной шкале (ромб ◆), не привязанная к длительности задачи.

### Просмотр

- **Gantt** — ромбы на шкале времени
- **Timeline** — полоска вех вверху страницы
- **Kanban** — блок «Upcoming milestones»

### Изменение даты

На диаграмме **Гант** перетащите ромб вехи влево/вправо. Изменение попадёт в локальный буфер; сохраните через панель внизу.

Новые вехи в текущей версии создаются при импорте данных или через API; в интерфейсе доступно редактирование даты существующих вех.

## 17. ИИ-ассистент

ИИ-ассистент помогает быстро получить ответы о **открытом проекте** без ручного поиска по таблице, Gantt или журналу аудита. Ответы формирует модель **GigaChat** на основе данных из базы (не из несохранённого локального буфера).

### Как открыть

1. Откройте проект (любое представление).
2. Нажмите кнопку **💬** в правом нижнем углу.
3. Введите вопрос в поле внизу панели и нажмите **Отправить** (или **Enter**; **Shift+Enter** — новая строка).

При первом открытии панели показываются **подсказки** — готовые вопросы, по клику подставляются в поле ввода.

### Что можно спрашивать

| Тема | Примеры |
|------|---------|
| Состав проекта | «Сколько задач в проекте?», «Какие задачи выполнены?» |
| Сроки | «Какие задачи в работе?», «Когда веха X?», «Просроченные задачи» |
| Этапы | «На каком этапе задача Y?», «Какие этапы завершены?» |
| Сдвиги дат | «Почему сдвинулся этап …?», «Когда менялись сроки задачи Z?» |
| Стоимость и трудозатраты | «Плановая стоимость по задаче …», «Фактические трудозатраты» |
| Организация | «Какие общие источники?», «Задачи релиза …», «Зависимости задачи …» |

Ассистент опирается на сохранённые в БД данные: статусы, даты, этапы, вехи, релизы, цели, общие источники, зависимости, стоимость, трудозатраты, пользовательские поля, **журнал изменений** и **комментарии** (в т.ч. к сдвигам этапов).

### Элементы панели чата

| Элемент | Назначение |
|---------|------------|
| Заголовок | Название проекта и модель GigaChat |
| **Очистить** | Сбросить историю диалога в этой сессии (данные проекта не меняются) |
| **×** (в шапке) | Закрыть панель |
| Поле ввода | Текст вопроса |
| **Отправить** | Запрос к ассистенту |

История диалога хранится **только в браузере** до закрытия вкладки или нажатия **Очистить**; на сервер уходит текущая переписка для контекста ответа.

### Если ассистент недоступен

Жёлтая плашка: *«GigaChat не настроен…»* — не задан API-ключ на backend. Администратору нужно:

1. Скопировать `.env.example` в `.env` в корне репозитория.
2. Заполнить `GIGACHAT_CREDENTIALS`.
3. Перезапустить: `docker compose up --build`.

Подробности — в [README.md](README.md#ии-ассистент-gigachat).

### Ограничения

- Ответы только по **текущему** проекту; другие проекты и внешние системы недоступны.
- **Несохранённые** правки (оранжевая подсветка, панель «Сохранить») ассистент не видит.
- Сдвиги этапов учитываются через **журнал аудита** и **комментарии**; старые изменения без записей в БД могут быть неизвестны ассистенту.
- Визуальные стрелки зависимостей, нарисованные только в браузере (`sessionStorage`), в контекст не попадают.
- Точность ответов зависит от полноты данных в карточках задач и от модели GigaChat; при нехватке фактов ассистент должен сообщить об этом.

## 18. Типичные сценарии

### Спланировать квартал

1. Откройте **Гант** с группировкой по категориям.
2. Расставьте/перетащите полосы задач.
3. Включите **Индикативные сроки** для предварительных оценок.
4. Добавьте комментарии к сдвигам при переносах.
5. Нажмите **Сохранить изменения**.

### Обновить статусы на стендапе

1. Откройте **Канбан**.
2. Перетащите карточки между колонками.
3. Сохраните изменения.

При входе в проект приложение может **автоматически предложить** перевести в «В работе» задачи с этапами, срок которых включает сегодня (см. [подсказку статуса](#подсказка-статуса-при-активном-этапе)).

### Приоритизировать бэклог

1. Откройте **Бэклог**, выберите метод (RICE / Ценность·Усилие / MoSCoW).
2. Заполните оценки в таблице или через drawer.
3. Сохраните — список пересортируется.

### Распределить задачи по релизам

1. Создайте релизы (**Релизы** в toolbar).
2. Откройте вкладку **Релизы**.
3. Перетащите задачи в нужные колонки.
4. Сохраните.

### Проверить, почему сдвинулся срок

1. Найдите красную/зелёную дугообразную стрелку сдвига на Gantt (у задачи, индикатива, этапа или вехи).
2. Включите **Комментарии к сдвигам** или откройте drawer → **История** / **Комментарии**.
3. При необходимости откройте **Журнал аудита** с фильтром «Даты».

### Зафиксировать сдвиг этапа

1. Откройте задачу в drawer → секция **Этапы**.
2. Нажмите **Сдвиг** у нужного этапа (или измените дату в списке).
3. Укажите новые даты и комментарий → **Сохранить сдвиг**.
4. На Gantt появится дуга сдвига; при влиянии на общий диапазон обновятся индикативные сроки.

### Завершить этап с проверкой дат

1. В drawer → **Этапы** → **Выполнено**.
2. Подтвердите или скорректируйте даты начала и окончания.
3. При изменении дат укажите комментарий → **Отметить выполненным**.
4. Процент задачи пересчитается; на Gantt появится зелёная полоса выполненного этапа.

### Добавить цепочку этапов из набора

1. Откройте задачу → вкладка **Этапы**.
2. В **Этап из шаблона** выберите **Загрузка детального слоя** или **Разработка Витрины**.
3. Нажмите **+ Добавить N этапов**.
4. При необходимости задайте сроки и нажмите **Запланировать** у каждого этапа.

### Удалить этап

1. В drawer → **Этапы** снимите отметку о выполнении, если этап завершён.
2. Нажмите **Удалить** у нужного этапа и подтвердите удаление.
3. Проверьте Gantt и таблицу: зелёная полоса и фактические даты задачи должны исчезнуть или обновиться по оставшимся этапам.

### Создать проект из Excel

1. Подготовьте файл **.xlsx** или **.xls** (или **.json**).
2. На стартовой странице перетащите файл в зону импорта.
3. Задайте название проекта и нажмите **Создать проект из файла**.
4. Откройте проект → **Таблица** — проверьте столбцы и задачи.
5. При необходимости добавьте этапы из шаблонов в drawer задач.

### Работа с общим источником

**Создать источник и привязать вторую витрину вручную:**

1. Откройте первую витрину → вкладка **Общая** → укажите **Источник** → **Сделать общим источником** (этапы и сроки переносятся в общий компонент).
2. Откройте вторую витрину с тем же **Источником** → **Привязать к общему источнику** (или выберите источник в списке → **Привязать**).
3. Альтернатива: **Источники** в toolbar → **Создать источник** → привяжите витрины из их карточек.

**Редактирование:**

1. Из любой привязанной витрины измените даты, статус или этапы — изменения сразу видны во всех использованиях.
2. В таблице у общего источника — бейдж **Общий**; в drawer — баннер со ссылками на другие витрины.

**Отвязка:**

1. В drawer нажмите **Отвязать (сохранить копию)** — витрина получит локальную копию и перестанет синхронизироваться.

### Спросить ассистента о проекте

1. Откройте проект и нажмите **💬** в правом нижнем углу.
2. Выберите подсказку или введите вопрос, например: «Какие задачи в работе?» или «Почему сдвинулся этап аналитики?».
3. При необходимости уточните в следующем сообщении (история диалога учитывается).
4. Для нового разговора нажмите **Очистить**.

## 19. Советы и ограничения

### Советы

- **Сохраняйте регулярно** — длительная работа без сохранения теряется при «Отменить» или закрытии вкладки.
- **Используйте таблицу** для массового ввода; **Gantt** — для визуальной расстановки сроков.
- **Swimlane + сворачивание** удобны для больших проектов с десятками категорий.
- **Звёздочка `*`** — быстрый признак несохранённой задачи в любом представлении.
- **Имя пользователя** важно для аудита — вводите осмысленное рабочее имя; комментарии и сдвиги этапов попадают в контекст ИИ-ассистента.
- **ИИ-ассистент** удобен для сводок и поиска по срокам/стоимости; перед важными решениями сверяйтесь с таблицей или Gantt.

### Ограничения

- Одновременное редактирование одной задачи несколькими пользователями может привести к конфликту версий.
- Создание новых вех через UI пока не реализовано (только перенос существующих на Gantt).
- Этапы сохраняются сразу (выполнение, сдвиг, добавление, удаление); остальные поля задачи — через панель сохранения.
- Удаление этапа доступно только для незавершённых; для выполненного сначала снимите отметку.
- Сдвиг этапа всегда требует комментарий; сдвиг задачи/вехи — комментарий необязателен, но рекомендуется.
- Шаблоны этапов проекта общие для всех задач этого проекта; стандартные шаблоны одинаковы во всех проектах.
- Удаление задачи из таблицы и из drawer — разные кнопки, но оба действия необратимы после подтверждения.
- ИИ-ассистент требует настроенный `GIGACHAT_CREDENTIALS` на сервере; без ключа чат только информирует об отсутствии настройки.
- Ответы ассистента не заменяют журнал аудита и не учитывают несохранённые локальные изменения.
